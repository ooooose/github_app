# TEST.md

## テスト方針

本プロジェクトでは、**過剰にならない範囲で重要なロジックを担保する**ことを目的にテストを実装しています。
すべてを網羅するのではなく、以下の観点で優先順位を付けています。

---

## 🎯 テスト対象の優先度

### ① ビジネスロジック（lib層）【最優先】

* APIレスポンス処理
* データ整形ロジック

**理由**

* 仕様変更や外部API変更の影響を受けやすい
* バグが発生するとユーザー影響が大きい

---

### ② UIコンポーネント（最低限）

* データが正しく表示されるか
* propsが反映されているか

**例**

* リポジトリ名が表示される
* Star数が表示される

---

### ③ テスト対象外

以下は保守コストを考慮しテスト対象外としています：

* CSS / Tailwindのスタイル
* レイアウト
* 細かいUIの見た目

---

## 🧱 ディレクトリ構成

テストは対象コードの近くに配置する「コロケーション」を採用しています。

```
src/
├── components/
│   ├── repo-item.tsx
│   └── __tests__/
│       └── repo-item.test.tsx
├── lib/
│   ├── github.ts
│   └── __tests__/
│       └── github.test.ts
```

---

## 🧪 使用技術

* Vitest
* Testing Library
* jsdom

---

### ✅ テストする例

* APIレスポンスが正しく処理される
* 検索結果が表示される
* 必須情報（リポジトリ名など）が描画される

---

### ❌ テストしない例

* ボタンの色
* margin / padding
* Tailwindクラス

---

## 🧪 テストコードの基本構造

```
describe("Component or Function", () => {
  test("期待される動作", () => {
    // arrange
    // act
    // assert
  })
})
```
